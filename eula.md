# GOE Messenger Viewer 이용약관

버전 2.4.4 | 최종 수정일: 2026-05-07

---

## 제1조  목적

본 소프트웨어는 GOE Messenger 사용자가 본인 계정으로 생성·보관한 백업 파일을
서비스 종료 또는 사용 종료에 대비하여 개인적으로 열람·보관할 수 있도록 지원하는
로컬 실행형 도구입니다.

감시, 조사, 징계, 평가, 분쟁 유도, 제3자 열람 제공, 편집, 재배포 등
목적 외의 사용을 허용하지 않습니다.

---

## 제2조  사용자 자격

본 소프트웨어는 오직 메신저 사용자 본인만 사용할 수 있습니다.
본인 계정으로 생성하여 본인이 적법하게 보관 중인 백업 파일에 대해서만
사용해야 하며, 타인의 데이터에 대한 사용은 허용되지 않습니다.

---

## 제3조  로컬 처리 원칙

복호화·변환·열람은 사용자의 로컬 기기 내부에서만 처리됩니다.
사용자의 대화 내용, 첨부 정보, 연락처, 생성된 뷰어 파일은 개발자 또는
제3자 서버로 자동 전송되지 않습니다.

---

## 제4조  버전 확인 및 텔레메트리 수집

본 소프트웨어는 실행 시 최소 버전 요건을 확인하고 익명 통계를 수집하기 위해
개발자 서버(Vercel + Supabase)에 아래 항목을 전송·기록합니다.

### 클라이언트가 전송하는 항목

| 항목 | 내용 |
|------|------|
| `version` | 소프트웨어 버전 문자열 |
| `device_id` | 설치 시 무작위 생성된 UUID (사용자 식별 불가) |
| `os` | OS 플랫폼 문자열 (예: `win32`) |
| `exe_hash` | 프로그램 무결성 검증값 (SHA-256) |
| `os_version` | OS 빌드 문자열 (예: `Windows-10-10.0.19045-SP0`) |
| `machine` | 머신 아키텍처 (AMD64 / ARM64 / x86) |
| `cpu_count` | 논리 CPU 코어 수 |
| `ram_total_mb` | 설치 PC 총 물리 메모리 (MB) |
| `ram_avail_mb` | 실행 시점 가용 메모리 (MB) |

### Vercel 서버가 자동 기록하는 항목

| 항목 | 내용 |
|------|------|
| `client_ip` | 접속 IP 주소 |
| `country` / `region` / `city` | 행정구역 정보 |
| `latitude` / `longitude` | 대략적 좌표 |
| `timezone` / `postal_code` | IANA 타임존 / 우편번호 |
| `continent` | 대륙코드 |

### 수집 목적

- 구버전 사용 차단 (보안)
- 프로그램 변조 탐지 (보안)
- 시스템 분포 익명 통계 → 프로그램 개선·기능 정책 결정

위 정보는 사용자를 특정하지 않고 통계적 정보로만 활용되며,
프로그램 개선 외 목적으로 사용되지 않습니다.

### 수집하지 않는 정보

- 메신저 대화 내용, 쪽지 본문, 첨부 파일
- 사용자 이름, 메신저 ID, 비밀번호
- 호스트네임, MAC 주소, 브라우저 쿠키

### 네트워크 미연결 시

전송 실패해도 소프트웨어는 계속 실행됩니다 (오프라인 보호).
서버가 `ok: false` 를 응답하는 경우(버전 만료) 소프트웨어가 종료됩니다.

---

## 제5조  금지행위

1. 타인의 백업 파일 또는 타인 계정 데이터에 사용
2. 생성 결과물의 업로드·공유·재배포·공개 게시·제3자 제공
3. 감시·조사·인사평가·징계·분쟁 유도 목적 사용
4. 보호조치 우회 또는 제3자 접근 허용
5. 소프트웨어 변조, 역컴파일, 소스코드 무단 추출

---

## 제6조  암호화 호환성 및 면책

GOE Messenger의 버전 및 환경에 따라 백업 파일의 암호화 방식이 다를 수 있으며,
본 소프트웨어가 지원하지 않는 방식의 경우 복호화가 불가능할 수 있습니다.
데이터 손실, 복호화 실패, 결과물 오류로 인한 손해에 대해 개발자는 책임을 지지 않습니다.
본 소프트웨어는 "있는 그대로(As-Is)" 제공됩니다.

---

## 제7조  비밀번호 및 보관 책임

비밀번호는 사용자가 직접 설정·관리하며, 분실 시 개발자에게 복구를 요청할 수 없습니다.
생성 결과물은 제3자가 쉽게 접근할 수 없는 방식으로 보관해야 합니다.
